# Creative direction and content handoff for Bumble v0

## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Translate current Bumble product decisions into concrete creative deliverables that unblock UX copy, tutorial/help tone, and near-term content implementation for the bee-farming loop.
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Lock the Bumble v0 voice and scope boundaries from current product decisions. 2. Produce the missing microcopy and help/tutorial handoff needed for the current farm loop. 3. Define light content direction for NPCs and simple buildings so implementation can proceed without overdesign. 4. Research external bee / plant / citizen-science sources for a later content layer.
<!-- SECTION:PLAN:END -->

## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
Product constraints from founder Q&A:
- Canonical title is Bumble / Star Sailors: Bumble.
- Core framing is a cosy mobile farm sim for casual players.
- Planet systems and wider Star Sailors crossover stay hidden for now.
- NPCs are flavour names on orders, not deep character arcs yet.
- Biomes are future alternate farm sites with different visuals/properties, not current lore-heavy travel beats.
- Keep bee hatch moments small and level-up mostly as a UI effect.
- Help text should read like a manual, not a character voice.
- Order reactions should stay tight.
- Honey quality should be removed from player-facing copy for now.
- Max-level/endgame can continue indefinitely for now.
- Simple buildings are in scope.
- The differentiator is the bee / honey-centered identity, with citizen-science links reserved for future ecosystem connections.
<!-- SECTION:NOTES:END -->
